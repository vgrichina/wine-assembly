# Code Review Report: Wine-Assembly x86 Emulator

## 1. Summary
This report details the current state of the Wine-Assembly x86 emulator. Significant progress has been made on SEH dispatch, focus management, and core CPU instruction correctness. However, several resource leaks, implementation gaps in the decoder (FPU/SSE), and GDI/Renderer bridging issues remain as critical blockers for complex applications like Winamp, MSPaint, and Space Cadet Pinball.

## 2. Resource Leaks & Lifetime Management
Accumulation of leaked memory and table slots remains a primary concern for long-running applications.

### 2.1 Memory Leaks in `wsprintf` (src/12-wsprintf.wat)
*   **Decimal Formatting:** `write_uint` and `write_uint_w` allocate temporary buffers on the heap via `$heap_alloc` but never call `$heap_free`. 
*   **Impact:** Affects all apps (Notepad, Winamp, Pinball) formatting scores, coordinates, or UI text.

### 2.2 Window Table & Control State Leaks (src/09a-handlers.wat, src/09c-help.wat)
*   **`DestroyWindow` Slot Leak:** `$handle_DestroyWindow` fails to call `wnd_table_remove`, leading to "Table Full" errors in dynamic apps like Winamp or MSPaint.
*   **Control State Leak:** `wnd_table_remove` zeroes the window record but does not free the `state_ptr`. 
*   **HWND Recycling:** Simple incrementing counter will eventually wrap or collide.

### 2.3 Filesystem View & Mapping Gaps (lib/filesystem.js)
*   **`UnmapViewOfFile` Leak:** `fs_unmap_view` removes the view from the tracking map but fails to free the guest memory.
*   **Missing Reference Counting:** Violates the Win32 memory-mapped I/O contract where changes in one view must be visible in others.

## 3. Implementation Gaps

### 3.1 x86 Decoder & Pipeline Limitations (src/07-decoder.wat)
*   **16-bit Addressing:** `0x67` prefix unimplemented.
*   **Far Control Flow:** `CALL far` and `JMP far` unimplemented.
*   **Group 5 POP:** `0xFF /7` (`POP m32`) crashes in the memory-target case.
*   **SSE/MMX Gaps:** Opcodes for SSE3/SSE4 and `EMMS` (`0x0F 0x77`) are missing. 
*   **Impact:** Multimedia apps (Winamp) get stuck in heavy FPU math (color quantization) due to these gaps.

### 3.2 Messaging & Windowing
*   **Asynchronous `SendMessage`:** `wnd_send_message` uses `PostMessage` semantics for x86 targets.
*   **Impact:** Breaks state machines relying on synchronous message returns (e.g., Pinball attract mode).
*   **Z-Order & Focus:** Does not handle "Always on Top" (`WS_EX_TOPMOST`) correctly.

### 3.3 Renderer & GDI Bridge (lib/renderer.js)
*   **Rectangular Clipping Only:** Complex regions (`SetWindowRgn`, `SelectClipRgn`) are ignored.
*   **Impact:** Winamp appears as a gray rectangle instead of its intended "skinned" shape.
*   **Bridged UI Controls:** `TreeView` is implemented in WAT but lacks a JS bridge.
*   **Font Scaling:** Fixed DLU scaling causes layout breakage on different resolutions.

## 4. Win32 API Gaps & Silent Stubs
Over 100 Win32 APIs remain stubbed with `$crash_unimplemented`.

### 4.1 Shell & GDI Stubs
*   **Shell Integration:** `ShellExecuteA/W` and `SHGetFileInfoA/W` are stubs.
*   **Advanced GDI:** `CreateDIBitmap`, `StretchDIBits`, and `SetDIBitsToDevice` lack proper ICM or color-space handling.
*   **Impact:** MSPaint cannot load/save images properly; Winamp cannot open URLs or retrieve icons.

### 4.2 System & Synchronization Gaps
*   **Sync Primitives:** `WaitForMultipleObjects` is unimplemented.
*   **Impact:** Multi-threaded apps (Winamp, Pinball) often hang or busy-wait.

## 5. CPU & OS Feature Progress
*   **Implemented:** SEH Dispatch (`RaiseException`) now walks the FS:[0] chain.
*   **Implemented:** `DestroyWindow` now correctly transfers focus to the promoted `main_hwnd` (Critical for Pinball).
*   **Fixed:** FPU `frndint` now honors the control word's rounding bits (Corrects PINBALL.DAT loading).
*   **Fixed:** `StretchDIBits` sub-rect blits now work correctly (Flippers/Ball animate in Pinball).
*   **Fixed:** `ADC`/`SBB` flags preserved correctly using raw mode.
*   **Outstanding:** `SETcc` reported to clobber full register when targeting memory (requires final verification).

# Wine-Assembly: Codebase Review Report (2026-04-25)

## 1. Resolved Issues (Audit 2026-04-25)

### CPU Emulator & ALU
*   **SHLD/SHRD Flag Updates:** [FIXED] 16-bit and 32-bit versions now correctly call `set_flags_shift`.
*   **BT/BTS/BTR/BTC Flagging:** [FIXED] Now correctly calls `set_cf_bit` to set Carry Flag based on original bit value.
*   **Far Call/Jump Support (0x9A, 0xEA):** [FIXED] Decoder now handles these as near CALL/JMP to offset (flat-mode), ignoring the selector.
*   **Inconsistent #DE Exception Codes:** [FIXED] Division errors now consistently raise `STATUS_INTEGER_DIVIDE_BY_ZERO` (0xC0000094).

## 2. Pending CPU/ALU Gaps (src/05-alu.wat, src/07-decoder.wat)

### MOVSX/MOVZX Coverage
*   **Issue:** Some variants use generic decoder patterns that might be less efficient than dedicated handlers.

## 3. Direct3D Immediate Mode (src/09aa-handlers-d3dim.wat, src/09ab-handlers-d3dim-core.wat)

### State Kind Opcode Swap
*   **Issue:** Line 724 of `src/09ab-handlers-d3dim-core.wat` has a comment swap: `7=render, 8=light` vs actual code `7=light, 8=render`.
*   **Impact:** Minor developer confusion.

### Limited Vertex Support
*   **Issue:** `d3dim_draw_primitive` only supports `D3DVT_TLVERTEX` (transformed and lit).
*   **Impact:** Most games that rely on the pipeline to transform or light vertices (`D3DVT_VERTEX`, `D3DVT_LVERTEX`) will not render.

### Missing D3D Lighting
*   **Issue:** `d3dim_exec_process_vertices` supports transformation but lighting logic is currently a no-op stub.

## 4. FPU Implementation Gaps (src/06-fpu.wat)

### Missing State Management
*   **Issue:** `FLDENV`, `FNSTENV`, `FRSTOR`, and `FNSAVE` (Groups 1 and 5 in `fpu_exec_mem`) call `fpu_crash_op`.
*   **Impact:** Applications that save/restore FPU state will crash.

### No BCD Support
*   **Issue:** `FBLD` and `FBSTP` (BCD load/store) are unimplemented.

## 5. JavaScript Framework & Host Imports

### VirtualFS Handle Leak
*   **Issue:** In `lib/filesystem.js`, `closeHandle` marks file handles as `closed: true` but never deletes them from the `this.handles` Map.
*   **Impact:** Potential memory leak in long-running sessions.

### Clipping Even-Odd Bug
*   **Issue:** `lib/host-imports.js` (line 923) uses a simple even-odd rule for child window clipping, lacking support for partial overlaps.
*   **Impact:** Visual artifacts in complex overlapping UI layouts.

### StretchDIBits Positioning/Clearing Bug
*   **Issue:** A debug override `SDB_FULL_BLIT` exists because of an unresolved bug where sub-rect blits or canvas clearing between calls produces incorrect output.

### DllMain Trap Masking
*   **Issue:** `lib/dll-loader.js` masks certain traps in `DllMain`. If the guest code triggers a trap but `EAX` is 1, the loader assumes success.
*   **Impact:** May hide legitimate initialization crashes.

### Incomplete CRT Locale/Char Support
*   **Issue:** `lib/host-imports.js` and `src/09a6-handlers-crt.wat` lack full support for DBCS lead bytes and multiple locales.
